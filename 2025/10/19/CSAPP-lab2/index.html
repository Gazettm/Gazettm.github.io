Jump

汇编如下

cmp指令为比较。带符号的如下。在linux中采用AT&T风格，前面为源操作数，后面为目的操作数。同时，我们知道参数有相应寄存器存储，第一个在rdi，第二个在rsi。所以此处为x < y得到 SF ≠ OF

CMP结果 标志位
目的操作数 ＜ 源操作数 SF ≠ OF
目的操作数 ＞ 源操作数 SF = OF
目的操作数 = 源操作数 ZF = 1

同时有下表

Instruction Jump Condition Description
jg Label ~(SF^OF)&~ZF >
jge Label ~(SF^OF) >=
jl Label SF^OF <
jle Label (SF^OF) or ZF <=

此时SF≠OF，异或之后为1，非之后为0。也就是说当x<y的时候会进行跳转操作，跳转到1145处进行y-x之后ret。当x>y的时候不跳转，会进行x-y并ret

cmov

汇编如下

这个是直接算出来y-x与x-y，之后cmp得到标志码之后，使用cmov这个条件移动指令。

Instruction Move Condition Description
cmovg S,R ~(SF^OF)&~ZF >
cmovge S,R ~(SF^OF) >=
cmovl S,R SF^OF <
cmovle S,R (SF^OF) or ZF <=

通过比较rdx(rsi,y)与rax(rdi,x)中的值，即比较x>y若成立，将rdx中的值移动到rax中，即 y-x 变为 x-y。

性能

现代处理器通过流水线来获得高性能。当遇到跳转指令时，处理器会根据分支预测器来猜测每条跳转指令是否执行。当发生错误预测的时候，会浪费大量的时间，导致程序性能严重下降。
