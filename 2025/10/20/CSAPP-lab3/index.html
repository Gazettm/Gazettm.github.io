CSAPP_lab3

loop

汇编如下

data16 cs nopw 0x0(%rax,%rax,1)

这一段比较特殊用于对齐，不讨论。

首先参数n存储在rdi中。吧0x1进入eax寄存器中，然后把rdi乘入rax中，再n-1，检查rdi是否大于1，再根据标志码寄存器的值判断是否jg跳转到1150位置。循环本质上就是通过条件测试与跳转的结合来实现的。

switch

汇编如下

(此处使用的是int所以是edi)将edi移入eax中，比较day是否大于7，这里考虑了默认范围，若不在默认范围直接返回了0。

这里则是switch的核心部分跳转表。day作为索引存入edx，lea指令加载跳转表基地址到rcx中，计算公式为

1152: 48 8d 0d ab 0e 00 00 lea 0xeab(%rip),%rcx 为rip当前指令下一个的地址值 加上0xeab，即为0x1159 + 0xeab = 0x2004，0x2004为跳转表基地址，被加载到rcx寄存器中

movslq从表中读取了偏移量，从rcx + rdx*4读取一个32位地址的值，并扩展64位到rdx中，得到目标相对偏移量。

之后相对偏移量 + 基地址rcx中的值得到绝对地址，之后跳转到绝对地址。

跳转表在0x2004，内容可能是这样的：

地址 day值 偏移量 目标地址 对应case
0x2004 0 0xFFFFFFXX 0x1162 default
0x2008 1 0x000000XX case1地址 case 1
0x200C 2 0x000000XX case2地址 case 2
… … … … …
0x201C 7 0x000000XX case7地址 case 7

根据跳转表跳转的地址来处理eax中的值并返回。

此处由于在case day的情况下返回了day，所以被优化了，从跳转表来看

拆开来看

可见day为1到7的时候，被偏移量相同0x2004 + 0xfffff163 = 0x2004 - 0xe9d = 0x1167

跳转到1167，即为ret指令。
