源代码

gomokuboard.h

底层的棋盘逻辑，定义了GomokuBoard类

private

私有成员中包含了一个QVector<QVector<Piece>> m_board;，用于储存棋盘信息，使用QVector来存每个位置的棋子类型。

int m_size;int类型的变量用于储存棋盘的大小

public

棋盘逻辑的核心方法

enum Piece { Empty, Black, White };定义了enum的枚举类型变量，用于判断棋盘某个位置是空/白棋/黑棋。

bool placePiece(int x, int y, Piece piece);用于落子，piece为落子类型，x，y为位置。

bool checkWin(int x, int y) const; 用于检查胜利，遍历四个方向若五子连珠则胜利。

访问器

int size() const { return m_size; }返回棋盘大小。

Piece pieceAt(int x, int y) const { return m_board[x][y]; }返回棋盘某位置的状态

gomokuboard.cpp

用于重置棋盘，使用了一个迭代器将每行每列的棋子状态为空。

用于落子，并返回一个bool值，用于后续判断落子是否成功。

用于判断是否五子连珠

定义一个向量组，使用迭代器 &dir来进行正向反向的检测，分别检测了x轴，y轴，主对角线和副对角线上的棋子是否五子连珠。

aiplayer.h

仅提供方法的工具类。用于给gamewindows文件提供ai的逻辑与落子。

因为需要读取棋盘状态所以#include "gomokuboard.h"

aiplayer.cpp

使用到随机数，因此需要包含#include <QRandomGenerator>

使用到abs函数，所以需要包括#include <cmath>

位置评估函数，为ai逻辑的核心内容。检测连子方法为与gomokuboard类中的checkwin函数相同，使用向量与迭代器来遍历。然后可以根据连子的多少来进行对位置重要性的评估。用变量score来进行量化。

使用Qvector存储QPoint类型的变量，用于存储所有最高分的位置。

QRandomGenerator::global()使用 Qt 全局的随机数生成器。

bounded(bestMoves.size())生成一个范围在 [0, bestMoves.size() - 1] 的随机整数（左闭右开区间）。

这样就实现了随机选择一个最优位置来帮助ai进行落子。

gamewindow.h

private

包含了棋盘逻辑，ai的工具类，当前的棋子与游戏模式成员。

还有两个用于绘制棋子和棋盘的两个函数。

protected

包含了两个Event，一个为绘图用，一个为鼠标点击操作用。

public

~GameWindow() override = default;基类为QMainWindow，加上override关键字

explicit GameWindow(QWidget *parent = nullptr);防止自动的类型转换。QWidget *parent = nullptr为Qt中的窗口构造参数

void ShowWinner(GomokuBoard::Piece winner);用于弹出获胜窗口

void exitGame();退出游戏的函数。

gamewindows.cpp

游戏的核心。

用于在开始时初始化一些东西并弹出对话框，赋值给游戏模式成员，用于接下来的判断。

用于最后弹出对话框，等待点击之后。点击之后调用exitGame函数，退出游戏。

绘画事件，用于画图。

鼠标操作事件，在鼠标操作时会调用。

人机对战且不是玩家回合时忽略点击

在玩家对战时每次切换当前棋子的类型并在之后检测胜利条件并更新棋盘绘图状态。

在人机对战时，人及放置然后检查胜利状态，更新之后切换到玩家操作。

总结

该程序分为了三个大类，每个类都有自己的功能。

优化方向：音效，界面自定义，ai逻辑。
