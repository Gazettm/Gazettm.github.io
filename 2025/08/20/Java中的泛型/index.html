Java中的泛型

Item 是一个类型参数（Type Parameter），它的作用是让这个 GenericStack 类可以处理任何类型的数据，而不需要为每种数据类型（比如 String, Integer, Person 等）都重新写一个单独的栈类。

方便

在创建和使用时会更加方便

这里声明了一个String类型和一个Integer类型的变量，可见十分方便，不需要重复为不同的数据单独写类。

类型安全

在之前，java使用的是Object 类来实现“通用”容器，因为所有类都继承自 Object。但这样取出数据时类型为Object，必须手动将它强制转换成你期望的类型。如果记错了类型，就会在运行时抛出 ClassCastException。

而在泛型中，类型错误在编译时就会被发现，而不是在程序运行时才崩溃。编译器知道你创建的 stringStack 只能存放 String，如果你尝试 push 一个 Integer，代码根本无法通过编译。

消除强制转换

从泛型栈中取出元素时，你不需要手动做强制类型转换。编译器知道 first.item 的类型就是 Item，而当你用 GenericStack<String> 时，Item 就是 String，所以 pop() 方法返回的直接就是 String。

总结

所以，Item 在这里是一个类型占位符。它在类被定义时是未知的，直到客户端代码创建这个类的对象时（如 new GenericStack<String>()），才被具体的类型（如 String）所替换。这个过程保证了类型安全，又提供了极大的灵活性。
