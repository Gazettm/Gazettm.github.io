何为装饰器

形如这样，接受一个函数，返回一个函数并在该函数身上加入一些额外的东西，这样可以让我们在不改动函数源码的情况下，动态调整函数的功能。

解释

  1. 装饰器顺序：从下往上应用，最内层装饰器最先执行
  2. 属性访问：只能访问最外层装饰器的属性，除非保存中间引用
  3. 引用覆盖：每次重新赋值都会丢失前一个装饰器层的引用

也就是说每次声明之后，属性就会变化。如果想要属性不变化，那么就必须使用from functools import wraps中的@warps来修饰修饰器中的内部函数。
