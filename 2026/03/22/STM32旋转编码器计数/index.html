GPIO

结构

GPIO接入PAB2总线，分为GPIOA和GPIOB等等，内部是寄存器和驱动器实现的。

八种输入输出

GPIO全称为General Purpose Input Output，有八种输入输出方式。

上拉输入：默认高电平

下拉输入：默认低电平

浮空输入：不确定

模拟输入：GPIO失能，引脚接入内部ADC

开漏输出：输出电平引脚，高电平为高阻态，低电平接VSS（驱动能力弱，依靠上拉电阻来输出高电平（在同时需要输入和输出时会用到

推挽输出：输出电平引脚，高电平为VDD，低电平接VSS

复用开漏输出：由片上外设控制，高电平为高阻态，低电平接VSS

复用推挽输出：由片上外设控制，高电平为VDD，低电平接VSS

EXTI外部中断

中断系统核心逻辑

中断机制的核心与 CSAPP 中讲解的 CPU 异常 / 中断模型一致：CPU 在执行主程序过程中，若触发特定中断条件（异步事件，如外设请求、引脚电平变化），会暂停当前执行流，转去执行预设的中断服务函数（异常处理程序），处理完成后恢复到原断点继续运行。

中断的核心特性包括：

优先级仲裁：多个中断源同时请求时，CPU 按中断优先级高低决定处理顺序；

中断嵌套：若当前中断服务函数执行时，有更高优先级的中断源请求，CPU 会暂停当前中断处理，转去处理高优先级中断，处理完成后再回到原中断继续执行。

STM32 中断核心特性

外部中断（EXTI）：

STM32 提供 EXTI0~EXTI15 共 16 条外部中断 / 事件线，每条中断线对应一组 GPIO 引脚：EXTI 线 n 对应所有 GPIO 端口的第 n 号引脚（如 EXTI0 对应 PA0、PB0、PC0…PG0，EXTI1 对应 PA1、PB1…PG1）。

当某一 GPIO 引脚触发电平变化时，需通过 AFIO（Alternate Function I/O，复用功能 I/O）的 EXTI 配置寄存器，判断具体是哪个端口的引脚触发了中断，再将中断请求传递给 EXTI 控制器。

NVIC 统一管理：

STM32 的所有中断（包括 EXTI、定时器、串口等）由内核级的 NVIC（Nested Vectored Interrupt Controller，嵌套向量中断控制器）统一管理：

每个中断通道支持 0~15 共 16 级可编程优先级；

优先级可分为 “抢占优先级” 和 “响应优先级”：抢占优先级决定中断嵌套能力（高抢占优先级可打断低抢占优先级的中断），响应优先级用于同抢占优先级的中断排队；

优先级分组（如 NVIC_PriorityGroupConfig）决定抢占优先级和响应优先级的位数分配（如分组 2：2 位抢占优先级 + 2 位响应优先级）。

比喻下来就是，AFIO是前台的接线员，知道是哪一个I/O口发出的信号，EXTI是前台，负责给NVIC通知，NVIC是经理，而CPU是员工。

旋转编码器的相关解释

这里使用了旋转编码器，旋转编码器的接入A，B两个触点，分别接入PB0和PB1。当向右正转的时候，A触点先响应，B随后来，这里设置的是EXTI下降沿触发中断，因此A下降沿的时候，B还处于低电平的时候时反转。同理，当B下降沿时，A还处于低电平时是正转。

这里只使用EXTI0配置中断也可以，只判断A引脚，当A引脚下降沿的时候，只看B就可以判断方向，但要注意的是，当正转的时候，B未转到位，当反转的时候，B已经转到位。

其他

EXTI_GetITStatus(EXTI_Line1) == SET 对这个进行判断，其中一个原因是STM32f103c8t6的9到5用的是同一个中断服务，15到10也是用的一个中断服务，所以这里要判断，当作一个规范，避免其他操作导致的中断运行了这个中断函数。
